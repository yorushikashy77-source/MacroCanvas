# MacroCanvas

MacroCanvas 是一个面向 Windows 的 PySide6 桌面宏映射工具。它可以把键盘、鼠标和窗口上下文组合成可复用的映射与宏动作，并通过 Kanata、winIOv2 或 Interception 处理输入输出。

## 主要能力

- 键盘与鼠标映射：按键、鼠标点击、滚轮、鼠标移动和等待动作。
- 宏动作编辑：支持动作树、循环动作、条件/否则分支、等待条件、子宏调用、拖拽排序和动作级测试。
- 可复用预设：可声明按键、整数和时长变量，将变量绑定到动作字段，并在每次子宏调用时覆盖默认值。
- 安全模拟预览：在不发送任何键鼠输入的前提下，估算单轮/总时长并展示动作时间线。
- 可视化运行调试：支持会话断点、下一动作暂停、单步、继续、动作定位，并显示子宏调用路径、实际参数与结束原因。
- 快速管理：映射与预设可按名称、快捷键、目标和状态搜索筛选，并批量启用或停用筛选结果。
- 系统托盘：关闭主窗口时可继续在托盘运行，并可显示窗口、暂停映射、停止宏或安全退出；录制或测试倒计时期间会优先进入可见的安全关闭流程。
- 多种执行模式：执行一次、固定次数、按住循环、开关循环和无限循环。
- 录制与整理：录制键鼠输入，自动整理等待、滚轮和坐标动作。
- 配置档案：根据前台进程、窗口标题或匹配条件自动切换配置。
- 配置安全：原子保存、配置备份、包含优先级顺序的差异预览、按区块选择性恢复、损坏配置恢复和导入回滚。
- 输入后端：普通模式使用 Kanata/winIOv2，游戏模式可使用 Interception。
- 运行安全：统一操作状态守卫、单实例锁、前台窗口隔离、输入释放保护和协调式退出。
- 隐私诊断：可导出不含原始配置，并对窗口、进程、快捷键、硬件 ID 和本地路径脱敏的诊断包。

## 流程控制与可复用预设

在预设卡片上点击 `动作` 可打开动作编辑窗口。普通动作可以拖入其他动作或固定分支下形成多级结构；选中一个条目后，再使用窗口上方的添加按钮，会把新动作放到相应层级。

### 条件分支和等待条件

- `＋ 条件` 会创建“条件成立”和“否则”两个固定分支。选择要判断的键盘或鼠标输入以及“按住时”或“松开时”，然后分别选中两个分支添加动作。
- `＋ 等条件` 会暂停当前宏，直到指定输入达到所选状态。超时后会结束当前宏；超时设为 `0 ms` 时不设超时。
- 分支内仍可继续添加条件、等待、子宏或普通动作，也可以通过拖拽调整同层顺序。

### 调用另一个预设

1. 在同一配置方案中先创建一个准备复用的目标预设。
2. 打开调用方预设的动作窗口，点击 `＋ 子宏`。
3. 在目标列选择目标预设，并设置调用次数与执行速度。
4. 先使用 `◇ 模拟` 检查时间线，再应用配置并进行实际测试。

当前预设不会出现在自己的子宏目标列表中。保存时也会检查间接循环调用和已经不存在的目标，避免递归执行或悬空引用。子宏速度会与调用方当前速度组合计算，并被限制在程序支持的范围内。

### 为子宏定义和传递变量

变量适合把“动作结构相同、只有按键或次数不同”的多个预设合并成一个可复用预设。支持的变量类型如下：

| 类型 | 默认值范围 | 常见绑定字段 |
| --- | --- | --- |
| 按键 | 程序支持的键盘键、鼠标键或滚轮输入 | 点击目标、条件按键 |
| 整数 | `1`–`100000`，具体动作仍会应用更严格的范围 | 滚轮格数、循环次数、调用次数、执行速度 |
| 时长 | `0`–`600000 ms`，部分保持/等待字段最小为 `1 ms` | 按住时长、等待时长、超时时间、循环间隔 |

以“调用同一套动作，但使用不同目标键”为例：

1. 打开目标预设的动作窗口，点击 `定义变量`，添加一个“按键”变量，例如 `目标键`，默认值设为 `A`。
2. 选中目标预设里的键盘动作，点击 `动作变量`，把“目标按键”绑定到 `目标键`。绑定成功的动作会显示变量标记。
3. 回到调用方预设，选中“调用子宏”动作并点击 `动作变量`。
4. 勾选 `覆盖默认值`，将 `目标键` 改为本次调用需要的按键；未勾选的变量继续使用目标预设的默认值。

编辑器只会列出类型和动作取值范围兼容的变量。例如，键盘点击的目标不会接受鼠标键变量值，执行速度也会限制在该字段允许的范围内。重命名或删除变量后，程序会清理失效的动作绑定和子宏覆盖值；修改完成后仍应重新模拟并应用配置。

### 使用运行调试器

可通过 `设置 → 运行调试器` 打开调试窗口，也可以在动作编辑窗口选中动作后点击 `断点`。断点只在本次程序会话中保留，不会写入配置文件；设置断点时会自动打开调试器并启用调试。

- 动作树中的 `●` 表示已设置断点，`▶` 表示当前动作，`◇` 仍表示变量绑定或子宏覆盖。
- `下一动作暂停` 会让所选宏在下一个动作发送输入前安全暂停。
- 命中断点后，`单步` 会执行当前动作并在后续动作前再次暂停；`继续` 会运行到下一个断点或结束。
- `定位动作` 会在用户主动点击后打开相应预设的动作窗口并选中当前动作；命中断点本身不会抢占目标程序焦点。
- 调试器会显示当前子宏调用路径、本次解析后的命名参数、程序持有的输入，以及正常完成、用户停止、条件超时、子宏不可用、运行超限或释放失败等结束原因。

调试暂停复用普通宏暂停的安全释放机制：程序会先释放当前持有的键鼠输出，继续时再恢复仍应保持的输出。如果在断点处关闭调试器，已经暂停的宏不会被擅自恢复，可使用主界面的 `继续` 按钮处理；尚未触发的“下一动作暂停”会被取消。

## 运行环境

- Windows；程序入口会主动拒绝在其他系统上运行。
- Python 3.12 或更高版本，建议使用 64 位 Python。
- PySide6 6.6 或更高版本。
- 如果使用游戏模式，需要另外安装 Interception 驱动。
- Kanata 和相关组件需要单独准备，仓库不包含第三方运行时二进制文件。

## 快速启动

在仓库根目录打开 PowerShell：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python main.py
```

如果 PowerShell 阻止虚拟环境脚本，可以直接使用：

```powershell
.\.venv\Scripts\python.exe main.py
```

## Kanata 与 Interception 组件

程序会按以下顺序寻找组件目录：

1. `MACROCANVAS_KANATA_DIR` 环境变量指定的目录；
2. 配置中保存的 Kanata 目录；
3. 可执行文件旁边的 `kanata` 目录；
4. `%LOCALAPPDATA%\MacroCanvas\kanata`；
5. 旧版兼容路径 `E:\kanata`。

普通模式至少需要对应的 Kanata 可执行文件和配置运行时。游戏模式还需要 `interception.dll` 以及已安装的 Interception 驱动。组件文件应从对应项目的官方发布渠道获取，不要把未经确认的 DLL 或驱动提交到仓库。

官方地址也可以从程序菜单 `设置 → 官方组件下载` 直接打开：

- [Kanata 源码仓库](https://github.com/jtroo/kanata) · [Kanata Releases](https://github.com/jtroo/kanata/releases)
- [Interception 源码仓库](https://github.com/oblitum/Interception) · [Interception Releases](https://github.com/oblitum/Interception/releases)

## 配置和数据位置

用户数据默认保存在：

```text
%LOCALAPPDATA%\MacroCanvas\
├─ config.json          主配置
├─ components.json      Kanata 组件目录设置
├─ ui-settings.json     托盘等界面偏好
├─ config_backups\      配置快照
├─ diagnostic.log       诊断日志
└─ kanata*.log          输入后端日志
```

程序源代码目录不会保存用户配置。删除或移动源码不会自动删除上述用户数据。

## 开发与测试

安装开发依赖：

```powershell
python -m pip install -r requirements-dev.txt
```

运行全部测试：

```powershell
python -m pytest -q
```

只进行语法检查：

```powershell
python -m compileall -q main.py core config engine macro ui tests
```

测试配置已限制为 `tests/`，并排除 `旧版备份`、缓存、构建和发布目录，避免归档副本被重复收集。

## 打包

安装 PyInstaller 后执行：

```powershell
python -m pip install pyinstaller
pyinstaller MacroCanvas.spec
```

生成的程序位于 `dist\MacroCanvas.exe`。发布便携版时，请在可执行文件旁边准备 `kanata` 组件目录，并在目标 Windows 环境中验证输入后端、驱动和退出释放行为。

## 目录结构

```text
main.py                 程序入口和单实例控制
core/                   常量、路径和运行状态
config/                 配置模型、校验、保存和导入导出
engine/                 Windows 输入、Kanata 和 Interception 后端
macro/                  动作模型、录制整理和宏调度
ui/                     主窗口、编辑器、档案管理和运行生命周期
tests/                  回归测试、架构测试和安全测试
MacroCanvas.spec        PyInstaller 打包配置
pytest.ini              pytest 收集配置
```

## 使用注意

MacroCanvas 会监听并可能模拟键盘和鼠标输入。请只在自己拥有或明确获准控制的设备上使用，并在启用游戏模式、驱动或全局输入钩子前保存工作。修改预设后，建议先使用“模拟”检查时间线，再应用配置并使用测试功能确认实际动作。
